# วิธีย้ายระบบ TA Task Board ไปบัญชี Google ใหม่

ใช้เมื่อต้องการเปลี่ยนบัญชี (อีเมล) ที่เป็นเจ้าของ Google Sheet + Apps Script + ปฏิทิน "TA Tasks"
หลังย้าย event ในปฏิทินจะไปลงที่บัญชีใหม่ และข้อมูล task จะเป็นข้อมูลปัจจุบันที่ดึงจากระบบเดิม

> หลักการ: บัญชีไหนเป็นเจ้าของ Apps Script (Execute as: Me) event ทั้งหมดจะไปลงปฏิทินบัญชีนั้น

---

## ขั้นที่ 1 — ดึงข้อมูลปัจจุบันจากระบบเดิม (บัญชีเก่า)

เลือกวิธีใดวิธีหนึ่ง:

**วิธี A (แนะนำ — ได้ข้อมูลล่าสุดแน่นอน):** เปิด Google Sheet เดิม → เมนู **ไฟล์ (File) → ดาวน์โหลด (Download) → Comma-separated values (.csv)** จะได้ไฟล์ข้อมูลปัจจุบันทั้งหมด

**วิธี B:** เปิดหน้า TA Admin Task Board (ที่เชื่อม 🟢 อยู่) แล้วกดปุ่ม **💾 บันทึกไฟล์ offline** เพื่อดาวน์โหลด `tasks.js` ที่มีข้อมูลล่าสุด (ใช้เป็นข้อมูลอ้างอิง)

---

## ขั้นที่ 2 — สร้าง Google Sheet ใหม่ (บัญชีใหม่)

1. **ออกจากระบบบัญชีเก่า แล้วล็อกอินด้วยบัญชีใหม่** (หรือใช้ Chrome อีกโปรไฟล์)
2. เปิด https://sheets.google.com สร้างสเปรดชีตใหม่ ตั้งชื่อ เช่น `TA Task Board`
3. เปลี่ยนชื่อแท็บชีตเป็น **`Tasks`**
4. นำเข้าไฟล์ CSV จากขั้นที่ 1: **ไฟล์ → นำเข้า → อัปโหลด** → เลือก **แทนที่ชีตปัจจุบัน** + ตัวคั่น **คอมมา**
5. ตรวจหัวคอลัมน์ให้ครบ:

   ```
   id | title | category | status | type | due_date | time | location | description | event_id
   ```

6. **สำคัญ: ล้างค่าในคอลัมน์ `event_id` ให้ว่างทั้งหมด** (เก็บหัวคอลัมน์ไว้ ลบเฉพาะข้อมูลด้านล่าง)
   เพราะ event_id เดิมชี้ไปปฏิทินของบัญชีเก่า ต้องให้บัญชีใหม่สร้าง event ใหม่เอง

---

## ขั้นที่ 3 — ใส่โค้ดและขออนุญาตสิทธิ์ (บัญชีใหม่)

1. ใน Sheet ใหม่: **ส่วนขยาย (Extensions) → Apps Script**
2. ลบโค้ดเดิมทิ้ง แล้วคัดลอกเนื้อหาไฟล์ `Code.gs` (เวอร์ชันล่าสุด) มาวางทั้งหมด → บันทึก
3. ที่กล่องเลือกฟังก์ชันด้านบน เลือก **`authorizeCalendar`** → กด **Run** ▶
4. กดอนุญาตสิทธิ์: เลือกบัญชี → Advanced → Go to … (unsafe) → **Allow**
   (จะได้สิทธิ์ทั้ง Sheets และ Calendar + สร้างปฏิทิน "TA Tasks" ในบัญชีใหม่)
5. (เลือกทำ) เลือกฟังก์ชัน **`syncAllToCalendar`** → กด **Run** เพื่อสร้าง event ในปฏิทินให้กับ task ที่มีวันที่ทั้งหมดทันที

---

## ขั้นที่ 4 — Deploy เป็น Web App ใหม่ (บัญชีใหม่)

1. มุมขวาบน **Deploy → New deployment**
2. ⚙️ → **Web app**
3. ตั้งค่า: **Execute as: Me** / **Who has access: Anyone**
4. **Deploy** → คัดลอก **Web app URL** ใหม่ (ลงท้าย `/exec`)

---

## ขั้นที่ 5 — อัปเดต URL ในหน้าเว็บ

1. เปิดไฟล์ `0 - Tasks/ta-admin-task-board.html` ด้วยโปรแกรมแก้ข้อความ
2. หาบรรทัด `const API_URL = '...';` แล้ว **แทนที่ด้วย URL ใหม่จากขั้นที่ 4**
3. บันทึก → เปิดหน้า board ควรขึ้น **🟢 เชื่อมต่อ Google Sheets**
4. ถ้า host ผ่าน GitHub Pages อย่าลืม `git add -A && git commit && git push`

---

## ขั้นที่ 6 — เก็บกวาดและแชร์ (แนะนำ)

- **แชร์ปฏิทิน "TA Tasks" ของบัญชีใหม่** ให้ทีม: Google Calendar → ปฏิทิน "TA Tasks" → Settings and sharing → Share with specific people
- **ปิดของเก่า:** ที่บัญชีเก่า ไปที่ Apps Script → Deploy → Manage deployments → **Archive** deployment เดิม เพื่อปิด URL เก่าไม่ให้ใช้งานได้
- (เลือกทำ) ลบปฏิทิน "TA Tasks" เก่าในบัญชีเก่า เพื่อไม่ให้ event ซ้ำซ้อน

---

## สรุปไฟล์ที่เกี่ยวข้อง

| ไฟล์ | บทบาท | ต้องทำอะไร |
|------|--------|------------|
| `Code.gs` | โค้ด backend (+ ปฏิทิน) | วางใน Apps Script บัญชีใหม่ |
| `ta-admin-task-board.html` | หน้าเว็บ | แก้ `API_URL` เป็น URL ใหม่ |
| `SETUP-Google-Sheets.md` | คู่มือตั้งค่าตั้งต้น | อ้างอิงเพิ่มเติม |
| `tasks.csv` / `tasks.js` | ข้อมูลสำรอง (offline) | ไม่บังคับ — อัปเดตได้จากปุ่ม "บันทึกไฟล์ offline" |
